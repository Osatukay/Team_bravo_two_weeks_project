## CloudWatch Metrics Explanation
| Metric | What it Measures | What it Tells You About the System | Why it Matters for Reliability |
|------|------|------|------|
| `RequestCount` | The total number of requests sent to the load balancer. | Shows how much traffic the system is receiving over time. | Helps identify traffic spikes, peak usage periods, or sudden increases that could overload the backend server. |

| `TargetResponseTime` | The time it takes for the backend server (target) to respond to the load balancer. | Indicates how quickly the application server processes requests. | High response times may indicate server overload, slow database queries, or inefficient processing. |

| `HTTPCode_Target_5XX_Count`| The number of server-side errors returned by the backend target. | Indicates failures within the backend application or server. | A spike in 5XX errors suggests the backend is failing to process requests properly, which directly affects system availability. |

| `HTTPCode_Target_4XX_Count` | The number of client-side errors returned by the backend target (e.g. 401, 403, 404). | Indicates issues with user requests, authentication failures, or missing resources. | While usually not server failures, high levels may indicate broken API calls, routing problems, or authentication issues. |

| `HTTPCode_ELB_4XX_Count` | Client errors generated by the load balancer itself. | Occurs when the load balancer rejects a request before it reaches the backend. | Can indicate malformed requests, blocked traffic, or potential malicious activity. |

| `ActiveConnectionCount` | The number of open connections currently maintained by the load balancer. | Shows how many clients are actively connected at any given moment. | High numbers combined with slow responses may indicate congestion or overloaded backend services. |

| `NewConnectionCount` | The number of new connections established with the load balancer. | Shows how frequently new sessions are being created. | Sudden spikes can indicate traffic surges or potential bot activity. |

| `ProcessedBytes` | The amount of data processed by the load balancer. | Reflects the volume of data transferred through the system. | Helps detect heavy workloads such as image uploads or large data transfers that may affect performance. |

| `UnhealthyRoutingRequests` | Requests routed to targets that were marked unhealthy. | Indicates traffic reaching failing or unstable backend instances. | Suggests health checks may not be working properly or the backend service is unstable. |
